## 📝 示例 1：简单文档生成工作流

### 场景

用户需要生成一份技术方案文档，包括：
- 需求分析
- 技术方案
- 实现计划
- 风险评估

### 执行步骤

#### 步骤 1：用户向 Director 发送请求

```
用户：请帮我生成一份关于"用户权限管理系统"的技术方案文档。

要求：
- 包含系统架构设计
- 包含数据库设计
- 包含实现步骤
- 包含风险评估
- 最终输出为 Markdown 格式
```

#### 步骤 2：Director 创建任务并分析

```bash
# Director 在内部执行
python3 scripts/task_update.py create TASK-20260421-001 "生成用户权限管理系统技术方案文档"

python3 scripts/task_update.py progress TASK-20260421-001 \
  "正在分析需求，准备分配给 Manager" \
  "分析需求🔄|拆解方案|调用执行|汇总回复"
```

#### 步骤 3：Director 调用 Manager

```
📋 Director·任务分配
任务ID: TASK-20260421-001
目标: 生成用户权限管理系统的完整技术方案文档

子任务:
  1. 需求分析 — 整理系统需求，输出需求清单
  2. 架构设计 — 设计系统架构，输出架构图和说明
  3. 数据库设计 — 设计数据库模型，输出 ER 图和 SQL
  4. 实现计划 — 制定开发计划，输出时间表和分工
  5. 风险评估 — 识别风险，输出风险清单和应对方案

输出要求: 所有产物为 Markdown 格式，可以直接合并成一份完整文档
```

#### 步骤 4：Manager 拆解并分配 Worker

Manager 拆解为 5 个独立子任务，逐个调用 Worker：

**Worker 1 - 需求分析**
```
📮 Manager·执行指令
任务ID: TASK-20260421-001
具体任务: 完成用户权限管理系统的需求分析

要求:
  - 列出功能需求（用户管理、角色管理、权限管理等）
  - 列出非功能需求（性能、安全、可扩展性等）
  - 标注优先级和复杂度

输出要求: Markdown 格式，包含需求清单表格
```

**Worker 2 - 架构设计**
```
📮 Manager·执行指令
任务ID: TASK-20260421-001
具体任务: 设计用户权限管理系统架构

要求:
  - 绘制系统架构图（用 Mermaid）
  - 说明各模块职责
  - 说明模块间通信方式

输出要求: Markdown 格式，包含架构图和说明文字
```

**Worker 3-5 同理...**

#### 步骤 5：Worker 执行任务

**Worker 1 返回结果**
```
✅ Worker·执行结果
任务ID: TASK-20260421-001
状态: done
摘要: 需求分析完成。识别 8 个功能需求和 5 个非功能需求。其中用户管理和权限管理标记为高优先级。
产物路径: shared/TASK-20260421-001/interim/01-需求分析.md
```

#### 步骤 6：Manager 汇总结果

```
✅ Manager·执行汇总
任务ID: TASK-20260421-001

所有子任务已完成：

1. ✅ 需求分析
   产物: shared/TASK-20260421-001/interim/01-需求分析.md

2. ✅ 架构设计
   产物: shared/TASK-20260421-001/interim/02-架构设计.md

3. ✅ 数据库设计
   产物: shared/TASK-20260421-001/interim/03-数据库设计.md

4. ✅ 实现计划
   产物: shared/TASK-20260421-001/interim/04-实现计划.md

5. ✅ 风险评估
   产物: shared/TASK-20260421-001/interim/05-风险评估.md

可以合并为完整文档。
```

#### 步骤 7：Director 汇总并回复用户

```bash
python3 scripts/task_update.py done TASK-20260421-001 \
  "✅ 技术方案文档已生成。包含 5 个部分：需求分析、架构设计、数据库设计、实现计划、风险评估。所有产物已保存到 shared/TASK-20260421-001/"
```

**回复用户**：
```
✅ 技术方案文档已完成！

已生成以下内容：
1. 需求分析 — 8 个功能需求，5 个非功能需求
2. 架构设计 — 模块化设计，3 层架构
3. 数据库设计 — 8 张表，包含完整 SQL
4. 实现计划 — 4 周交付，3 人团队
5. 风险评估 — 5 个风险，5 个应对方案

所有文件已保存到: shared/TASK-20260421-001/interim/

你可以将这 5 个文件合并成一份完整的技术方案文档。
```

## 📊 示例 2：代码审查工作流

### 场景

```
用户：请审查我的用户认证模块代码，检查：
- 代码质量
- 安全问题
- 性能优化
- 测试覆盖
```

### 任务分解

| 子任务 | Worker | 输出 |
|------|--------|------|
| 代码质量检查 | Worker A | 代码质量报告 |
| 安全扫描 | Worker B | 安全问题清单 |
| 性能分析 | Worker C | 性能优化建议 |
| 测试覆盖分析 | Worker D | 测试覆盖报告 |

### 整个流程的时间轴

```
T+0s   用户发送请求
T+1s   Director 创建任务 TASK-20260421-002
T+2s   Director 调用 Manager
T+3s   Manager 拆解为 4 个子任务
T+4s   Manager 逐个调用 4 个 Worker
T+5-30s Worker 并行执行（假设每个需要 5-10 秒）
T+35s  Manager 汇总 4 个结果
T+36s  Director 回复用户，提供完整的审查报告
```


### 预期指标

| 场景 | 任务数 | 预期耗时 | 瓶颈 |
|------|--------|---------|------|
| 简单 | 1 | < 5s | Worker 执行 |

- 并行化：Manager 同时调用多个 Worker

## 🔍 故障注入测试

### 测试 1：Worker 失败

```bash
# 模拟 Worker 执行失败
python3 scripts/task_update.py progress TASK-FAIL-001 \
  "执行失败" \
  "接收✅|执行❌"

# 验证 Manager 如何处理错误
# 预期：Manager 应该捕获错误并向 Director 报告
```

### 测试 2：中途任务中断

```bash
# 创建任务但不完成
python3 scripts/task_update.py create TASK-INCOMPLETE-001 "未完成的任务"
python3 scripts/task_update.py progress TASK-INCOMPLETE-001 "执行中..." "执行🔄"

# 等待 5 分钟，任务应该保持进行中状态
# 验证 tasks.json 是否能恢复
```

### 测试 3：并发写入冲突

```bash
# 同时运行两个脚本写入同一任务
for i in {1..5}; do
  python3 scripts/task_update.py progress TASK-CONCURRENT-001 \
    "并发测试 $i" &
done
wait

# 验证 tasks.json 是否保持一致（应该只有最后一条记录）
```
