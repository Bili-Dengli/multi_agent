## 一、tasks.json 存储方案

### 选定：✅ 选项 A - 符号链接

### 技术实现

**目录结构**：
```
~/.openclaw/workspace/multi_agents/
├── data/
│   └── tasks.json              # 任务状态文件（主存储）
└── shared/
    └── <task_id>/             # 任务产物目录
        ├── interim/           # Worker 中间产物
        └── final/             # Director 最终产物
```

**符号链接**：
```
~/.openclaw/workspace-director/data → ~/.openclaw/workspace/multi_agents/data
~/.openclaw/workspace-director/shared → ~/.openclaw/workspace/multi_agents/shared

~/.openclaw/workspace-manager/data → ~/.openclaw/workspace/multi_agents/data
~/.openclaw/workspace-manager/shared → ~/.openclaw/workspace/multi_agents/shared

~/.openclaw/workspace-worker/data → ~/.openclaw/workspace/multi_agents/data
~/.openclaw/workspace-worker/shared → ~/.openclaw/workspace/multi_agents/shared
```

**task_update.py 访问方式**：
```python
# 直接读写本地路径（因为符号链接指向同一文件）
TASKS_FILE = Path("data/tasks.json")
SHARED_DIR = Path("shared")
```

**Windows 兼容性**：使用 `os.path.realpath()` 解析符号链接，或使用 `pathlib.Path.resolve()`

## 三、Agent 注册方式

### 选定：✅ 选项 B - install.sh 自动注册

### 技术实现

**install.sh 功能**：
```bash
#!/bin/bash
set -e

EDICT_HOME=~/.openclaw

# 1. 创建数据目录
mkdir -p $EDICT_HOME/workspace/multi_agents/data
mkdir -p $EDICT_HOME/workspace/multi_agents/shared

# 2. 注册 3 个 Agent
openclaw agents add director \
  --workspace $EDICT_HOME/workspace-director \
  --subagents manager

openclaw agents add manager \
  --workspace $EDICT_HOME/workspace-manager \
  --subagents worker

openclaw agents add worker \
  --workspace $EDICT_HOME/workspace-worker

# 3. 创建符号链接
ln -sf $EDICT_HOME/workspace/multi_agents/data $EDICT_HOME/workspace-director/data
ln -sf $EDICT_HOME/workspace/multi_agents/shared $EDICT_HOME/workspace-director/shared

ln -sf $EDICT_HOME/workspace/multi_agents/data $EDICT_HOME/workspace-manager/data
ln -sf $EDICT_HOME/workspace/multi_agents/shared $EDICT_HOME/workspace-manager/shared

ln -sf $EDICT_HOME/workspace/multi_agents/data $EDICT_HOME/workspace-worker/data
ln -sf $EDICT_HOME/workspace/multi_agents/shared $EDICT_HOME/workspace-worker/shared

# 4. 复制脚本
cp scripts/task_update.py $EDICT_HOME/workspace-director/scripts/
cp scripts/task_update.py $EDICT_HOME/workspace-manager/scripts/
cp scripts/task_update.py $EDICT_HOME/workspace-worker/scripts/

echo "✅ Multi-Agents 安装完成"
```

**验证方式**：
```bash
openclaw agents list | grep -E "director|manager|worker"
```

## 五、任务 ID 命名规则

### 选定：✅ 选项 A - 时间戳 TASK-YYYYMMDD-NNN

### 技术实现

**格式**：`TASK-YYYYMMDD-NNN`

- `YYYYMMDD`：年年月月日日（例：20260421）
- `NNN`：当日序号，从 001 开始，自增

**生成逻辑**：
```python
from datetime import datetime

def generate_task_id():
    today = datetime.now().strftime("%Y%m%d")
    # 读取 tasks.json 获取当天的最大序号
    tasks = load_tasks()
    today_tasks = [t for t in tasks if t["id"].startswith(f"TASK-{today}")]
    if today_tasks:
        max_seq = max(int(t["id"].split("-")[-1]) for t in today_tasks)
        seq = max_seq + 1
    else:
        seq = 1
    return f"TASK-{today}-{seq:03d}"
```

**示例**：
```
TASK-20260421-001  # 2026年4月21日第一个任务
TASK-20260421-002  # 2026年4月21日第二个任务
TASK-20260422-001  # 2026年4月22日第一个任务
```

## 七、错误处理策略

### 选定：✅ 选项 B - 直接返回错误

### 技术实现

**错误传播流程**：
```
用户 → Director → Manager → Worker
  ↑         ↑          ↑
  └──────────┴──────────┘
           错误逐级上报
```

**错误返回格式**：
```
❌ 执行失败
任务ID: TASK-20260421-001
错误信息: [具体错误描述]
建议: [可选：如何修复]
```

**示例场景**：

1. **Worker 执行失败**
   ```
   ❌ Worker·执行失败
   任务ID: TASK-20260421-001
   错误信息: 数据库连接失败
   建议: 检查数据库配置
   ```
   → Manager 收到后 → 继续返回给 Director → Director 回复用户

2. **Manager 调用 Worker 超时**
   - 直接返回超时错误给 Director（不重试）

3. **Director 调用 Manager 失败**
   - 捕获异常 → 返回给用户 "任务处理失败，请重试"

**代码示例**：
```python
def call_subagent(agent, prompt):
    try:
        result = openclaw.call(agent, prompt)
        return result
    except Exception as e:
        return f"❌ 执行失败\n错误信息: {str(e)}"
```
