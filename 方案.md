# Multi-Agents 企业协作技能包

> **版本**：v2.0 - 企业级多 Agent 协作技能

## 一、概述

**multi-agents** 是一套基于企业组织架构设计的多 Agent 协作技能包，模拟真实企业的层级管理结构，通过**决策者、监督者、执行者**三层架构完成复杂任务的拆解、分配、执行与汇报。

### 核心目标

- 将复杂任务自动拆解为可并行执行的子任务
- 模拟企业组织层级，各角色职责明确
- 支持任务状态追踪、产物统一管理
- 可打包为独立 Docker 镜像部署

## 二、角色定义

### 2.1 Director（决策者 / Team Leader）

**定位**：相当于企业 CEO / 项目负责人，负责战略决策和整体把控。

**职责**：
- 接收用户的原始目标
- 分析任务性质，判断是否需要多 Agent 协作
- 制定整体执行策略（串行/并行/混合）
- 将任务拆分为任务包，分配给各 Manager
- 汇总最终结果并输出

**决策逻辑**：
```
IF 任务可在3步内完成 → 直接执行，不启动多Agent（快速模式）
IF 任务包含多个独立子目标 → 并行分配给多个Manager
IF 任务有严格先后依赖 → 串行分配给Manager
IF 任务混合依赖 → 混合模式
```

---

### 2.2 Manager（监督者）

**定位**：相当于企业中层管理者（部门经理），负责模块内的任务管理。

**职责**：
- 接收 Director 分配的任务包
- 将任务包拆解为具体操作步骤
- 为每个步骤分配一个 Worker
- 收集 Worker 结果，整合后汇报给 Director

**任务拆解规则**：
```
每个子任务应满足：
- 有明确的可验证输出
- 单个 Worker 可独立完成
- 预估执行时间 < 2分钟
- 不依赖其他未完成子任务（或依赖关系明确）
```

---

### 2.3 Worker（执行者 / Specialist）

**定位**：相当于企业一线员工 / 专业人员，负责具体操作执行。

**职责**：
- 接收 Manager 分配的具体任务
- 调用工具完成任务
- 将结果以结构化格式返回
- 如遇异常，明确描述失败原因后上报

**执行约束**：
```
- 不得自行扩大任务范围
- 失败时必须上报，不得自行掩盖
- 输出格式必须符合 Manager 的期望格式
```

## 三、协作规范

### 3.1 产物目录（必须）

```
~/.openclaw/workspace/multi_agents/sessions/<session_id>/shared/
├── <task-slug>/              # 任务英文名
│   ├── interim/            # 中间产物
│   │   └── 01-findings.md
│   └── final/              # 最终交付物
│       └── report.md
└── state.json              # 任务状态（可选）
```

> **核心原则**：产物散落、信息断裂是协作失败的根本原因。

---

### 3.2 Owner 原则（必须）

**每个文件同一时刻只有一个 owner**。

任务分配时明确：
```
任务包A:
  - Agent A 负责: api.ts
  - Agent B 负责: service.ts
  - 如需修改对方文件 → 发消息协商
```

---

### 3.3 消息传递

用 OpenClaw 内置的 `sessions_send` 或类似机制。

消息格式：
```
To: <Agent ID>
Content: <发现内容摘要>
Artifact: <产物路径>
```

---

### 3.4 状态流转

```
accepted → in_progress → done
                  ↘ blocked → 重试 → done
```

---

### 3.5 循环依赖处理

当 A 等 B、B 也等 A 时，降级为串行迭代：
```
A 出初稿 → B 审核提意见 → A 最终确认
```

## 四、执行流程

### 标准模式

```
用户请求
   │
   ▼
┌─────────────────────┐
│ Director 分析目标  │
│ - 判断任务复杂度   │
│ - 决定协作模式    │
└────────┬──────────┘
         │
    ┌────┴────┐
    ▼         ▼
直接执行    启动团队
             │
             ▼
         ┌───────┴───────┐
         ▼               ▼
    Manager A        Manager B
    (并行)          (并行)
         │               │
         ▼               ▼
    ┌────┴────┐    ┌────┴────┐
    ▼         ▼    ▼         ▼
   Worker   Worker  Worker   Worker
         │               │
         ▼               ▼
    汇总给 Director → 输出报告
```

# Multi-Agent Director

## 描述

你是多 Agent 团队的决策者（Team Leader），负责接收用户目标，制定协作策略，拆解任务并分配给监督者，最终整合结果输出。

## 能力范围

- 分析任务复杂度，判断是否需要多 Agent 协作
- 制定任务拆分方案（串行/并行/混合）
- 分配任务给各 Manager
- 汇总结果，输出最终报告
- 快速模式：简单任务直接执行

## 执行步骤

1. 接收用户目标，分析复杂度
2. 如果简单 → 直接执行，跳过下面步骤
3. 创建任务目录：
   ```bash
   mkdir -p shared/<task-slug>/{interim,final}
   ```
4. 拆解任务为任务包
5. 分配给 Manager（通过消息或直接调用）
6. 等待 Manager 汇报
7. 汇总结果，输出报告到 shared/<task-slug>/final/

## 协作规范

- 产物统一放 shared/<task-slug>/ 目录
- 每个文件指定明确的 owner
- 完成即回传，包含摘要 + 产物路径
```

## 六、Docker 镜像设计

### 6.1 Dockerfile

```dockerfile
# =============================================================================
# OpenClaw Skills - Multi-Agents Image
# =============================================================================
# 描述：多 Agent 协作技能镜像
# 包含：director, manager, worker
# =============================================================================

ARG BASE_IMAGE=alpine/openclaw:2026.3.13-1
FROM ${BASE_IMAGE}

USER root

# 复制技能文件到镜像
COPY skills/multi_agents/director/  /home/node/.openclaw/workspace/skills/director/
COPY skills/multi_agents/manager/   /home/node/.openclaw/workspace/skills/manager/
COPY skills/multi_agents/worker/    /home/node/.openclaw/workspace/skills/worker/

# 复制脚本和依赖（如有）
COPY skills/multi_agents/scripts/   /home/node/.openclaw/workspace/skills/multi_agents/scripts/
COPY skills/multi_agents/package.json /home/node/.openclaw/workspace/skills/multi_agents/

# 安装依赖
RUN npm install --prefix /home/node/.openclaw/workspace/skills/multi_agents \
    --registry https://registry.npmmirror.com

# 创建状态存储目录和共享目录
RUN mkdir -p /home/node/.openclaw/workspace/multi_agents/sessions \
             /home/node/.openclaw/workspace/multi_agents/shared

# 复制身份文件到 workspace 根目录
COPY skills/multi_agents/SOUL.md    /home/node/.openclaw/workspace/
COPY skills/multi_agents/IDENTITY.md /home/node/.openclaw/workspace/

RUN chown -R node:node /home/node/.openclaw

USER node
```

### 6.2 构建命令

```bash
# 构建镜像
./docker/build.sh --preset multi-agents

# 运行
docker run --rm -it openclaw-multi-agents:latest sh

# 挂载会话数据和共享目录（持久化）
docker run --rm -it \
  -v $(pwd)/sessions:/home/node/.openclaw/workspace/multi_agents/sessions \
  -v $(pwd)/shared:/home/node/.openclaw/workspace/multi_agents/shared \
  openclaw-multi-agents:latest sh
```

## 七、注意事项

1. **Token 消耗**：多 Agent 是单 Agent 的 3-4 倍，先确认任务值得拆分
2. **Owner 原则**：分配任务时指定文件 owner，防止冲突
3. **产物统一**：所有产物放 shared/ 目录，不要散落各处
4. **循环依赖**：遇到 A等B、B等A 时，降级为串行迭代
5. **Docker 基础镜像**：依赖 `alpine/openclaw:2026.3.13-1`
